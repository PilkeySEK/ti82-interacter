I wanted to upload programs in .82p or .82g format to a TI-82 but I didn't have a cable for it - only an ESP32. So I did what any programmer would do and reinvented the wheel. TiLP suprisingly does not cover this use case (it appears).

You need a .82g or .82p or other .82* file that you want to upload. The program expects the data to be sent over serial (e.g. /dev/ttyUSB0), which - notably - is NOT what most/all official or widely used cables do. What I do is that I connect a jack cable to the TI-82 and wire the tip, ring and ground to specific pins on an ESP32, which forwards data received and sent over the serial USB port. It should work if you have a similar setup.

**NOTE:** This was generated by Claude, I provided it some files which describe the TI-82 protocol and it generated the rust code for this, with not a lot of guidance. I originally wanted to write all of this myself but to be honest I was tired of the TI protocol which is like ... not that good ... so I just forced AI to do it :)
